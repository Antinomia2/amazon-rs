# Amazon Video Games Recommender Systems

## Overview

This project develops and compares recommender-system approaches on the `Video_Games` category of the Amazon Reviews 2023 dataset. The work combines collaborative filtering, content-based filtering, and user segmentation to study how user-item ratings and product title & description can be used to predict missing ratings and generate Top-10 recommendations for each user.

The source data comes from the McAuley Lab Amazon Reviews 2023 dataset on Hugging Face:

https://huggingface.co/datasets/McAuley-Lab/Amazon-Reviews-2023

The project focuses on the cleaned review and product-metadata tables produced during preprocessing, together with the filled user-item matrices generated by models trained on the dataset.

## Project Objectives

- Explore (EDA) the Amazon `Video_Games` review data before modeling.
- Build cleaned datasets containing the fields needed for recommendation.
- Implement collaborative filtering with KNN and matrix factorization.
- Implement content-based filtering using product `title` and `description`.
- Compare frequency-based and neural embeddings.
- Generate Top-10 recommendation for each user while excluding products already reviewed by the user.
- Segment users with K-Means over completed rating matrices.
- Evaluate the models using MSE and RMSE.

## Notebook roles:

- `main/analisi_esplorativa.ipynb`: loads the Amazon `Video_Games` data, performs exploratory analysis, creates derived variables for correlation analysis, cleans the review and metadata tables, and saves the cleaned CSV files.
- `main/cf.ipynb`: implements collaborative filtering with KNNBaseline and SVD, performs grid search and held-out evaluation, fills missing ratings, and displays Top-10 recommendations.
- `main/cb.ipynb`: implements content-based filtering with TF-IDF and neural embeddings, evaluates item-based KNN predictions, fills missing ratings, and displays Top-10 recommendations.
- `main/clustering_cf.ipynb`: clusters users from the collaborative-filtering filled matrices.
- `main/clustering_cb.ipynb`: clusters users from the content-based filled matrices.
- `graphs/`: contains the main plots generated from exploratory analysis and clustering.

## Data Preparation and Main Transformations

The exploratory notebook starts from the Amazon `Video_Games` review and metadata data, then reduces them to the fields needed by the recommender systems.

Main cleaning steps:

- Kept only `rating`, `parent_asin`, and `user_id` from reviews.
- Kept only `title`, `description`, and `parent_asin` from metadata.
- Removed metadata rows without a product title.
- Removed reviews whose `parent_asin` no longer appeared in the cleaned metadata.
- Saved the cleaned review and metadata tables as `reviews_clean.csv` and `meta_clean.csv`.


## Methodology

### Exploratory Data Analysis

EDA examines rating frequencies, review counts per user and product, review-image usage, helpful votes, and Spearman correlations between selected review and metadata variables.

The rating distribution is strongly skewed toward positive ratings, with 5-star reviews forming the largest group.

### Collaborative Filtering

The collaborative-filtering notebook builds a user-item rating matrix from `reviews_clean.csv` and evaluates two approaches with the Surprise library.

The first model is `KNNBaseline`. A 5-fold grid search tests:

- `k`: 15, 25, 35
- similarity: `msd`, `pearson`, `cosine`
- user-based and item-based configurations

The best KNN configuration is item-based KNN with `k = 25` and `msd` similarity. The notebook then evaluates the selected model on an 80/20 train-test split, fills the missing entries of the user-item matrix, and builds Top-10 recommendations for each user.

The second collaborative-filtering model is SVD matrix factorization. A 5-fold grid search tests combinations of:

- `n_factors`: 70, 100, 130, 160
- `n_epochs`: 20, 30, 40

The best SVD configuration uses `n_factors = 100` and `n_epochs = 30`. The trained model is used to fill the user-item matrix and generate Top-10 recommendations in the same way as KNN.

### Content-Based Filtering

The content-based notebook uses the product `title` and `description` fields from `meta_clean.csv`. These fields are merged into a single text representation for each item.

Two item-embedding strategies are implemented:

- TF-IDF embeddings with a custom tokenizer.
- Neural embeddings using `sentence-transformers/average_word_embeddings_komninos`.


For both embedding types, item-item cosine similarity is computed. Missing ratings are predicted with item-based KNN using the 10 nearest content neighbors. If a user has not rated any of those neighbors, the notebook falls back to the neighbors' product-level average ratings.

### Clustering / User Segmentation

User segmentation is performed after matrix filling because K-Means requires complete numeric vectors. The clustering notebooks normalize each completed user-item matrix and apply K-Means with cosine-based silhouette evaluation.

The number of clusters is selected from elbow plots:

| Matrix | Selected K | Silhouette |
| --- | ---: | ---: |
| Collaborative KNN | 6 | 0.3458 |
| Collaborative SVD | 3 | -0.0674 |
| Content TF-IDF | 4 | 0.4825 |
| Content Neural | 3 | 0.6583 |

### Comparison Between Methods

The notebooks compare the collaborative-filtering models with each other and the two content-based embedding strategies with each other. The final report also compares all four implemented methods using held-out MSE and RMSE.

The recommendation outputs are compared by merging Top-10 lists rank by rank, allowing inspection of how the recommended product titles differ between KNN and SVD, and between TF-IDF and neural embeddings.

## Evaluation

The project uses MSE and RMSE as the main predictive metrics.

Collaborative-filtering models are evaluated in two ways:

- 5-fold cross-validation during grid search.
- Final 80/20 train-test evaluation using the best configuration.

Content-based models are evaluated with an 80/20 train-test split using item-based KNN over the embedding similarity matrices.

Final held-out results reported by the project:

| Method | MSE | RMSE |
| --- | ---: | ---: |
| Collaborative Filtering - SVD | 0.932949 | 0.965893 |
| Collaborative Filtering - KNN | 1.037704 | 1.018678 |
| Content-Based - Neural Embedding | 1.266831 | 1.125536 |
| Content-Based - TF-IDF | 1.286656 | 1.134309 |

SVD gives the best predictive performance among the implemented methods. The neural content-based model performs slightly better than TF-IDF on MSE and RMSE, while the report notes that TF-IDF recommendations tend to be somewhat more varied and neural recommendations tend to be more semantically close.


## Main Results / Key Findings

- The full cleaned review data is extremely sparse: most users have very few reviews.
- Ratings are heavily skewed toward high scores; the cleaned review data has an average rating of about 4.05 and a median rating of 5.
- Item-based KNN performs better than user-based KNN in the collaborative-filtering grid search, which is consistent with the high sparsity of shared user histories.
- SVD matrix factorization achieves the strongest predictive performance among the implemented methods.
- Content-based filtering is less accurate on rating prediction than collaborative filtering, but it uses product text and can recommend from item similarity rather than relying only on shared rating patterns.
- In the content-based comparison, neural embeddings slightly outperform TF-IDF by MSE/RMSE, while TF-IDF produces somewhat more varied recommendation lists.
- Clustering is used descriptively to segment users by the completed rating vectors and summarize each cluster by user count, average number of reviews, and mean rating.
